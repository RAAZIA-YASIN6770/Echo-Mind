Change Control Process

1. Raise Change Request (CR)
   - Submit CR form: description, reason, impact, estimated cost, requested by
2. Assessment
   - PM + Technical Lead review impact on scope/timeline/cost
3. Approval
   - If cost or schedule impact > X, escalate to Sponsor for approval
4. Implementation
   - Approved CR moves to backlog with priority and target sprint
5. Communication
   - Stakeholders notified, documentation updated
6. Audit
   - Record CR decisions and approvals in project docs
